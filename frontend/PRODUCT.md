# Product

## Register

product

## Users

Trendsight is used by public-opinion analysts, emergency-response staff, communications teams, and course evaluators reviewing an intelligent public-opinion analysis platform. Users are usually scanning multiple events under time pressure and need to identify risk, sentiment, propagation, authenticity, and response priorities quickly.

## Product Purpose

Trendsight aggregates public-opinion events from news and social platforms, discovers hot topics, analyzes sentiment and propagation trends, evaluates risk and authenticity, and presents event-level analysis reports with intelligent Q&A. Success means a user can understand the current risk landscape, prioritize high-risk incidents, inspect a report, and configure monitoring sources without visual noise or workflow friction.

## Brand Personality

Calm, authoritative, analytical. The product should feel like a serious monitoring console from a mature public-opinion intelligence team, not a decorative student demo or generic SaaS landing page.

## Anti-references

- Toy-like sentiment glyphs, decorative timelines, and novelty visuals that slow scanning.
- Loud blue/purple gradients, glassmorphism, excessive glow, and generic AI-dashboard styling.
- Oversized hero typography inside authenticated tool screens.
- Repeated identical feature cards or card-heavy compositions that feel like templates.
- Visual clutter that exposes implementation concepts such as crawler pipelines before analyst-facing priorities.

## Design Principles

1. Analyst workflow first: global risk summary before filters, then event comparison, then report detail.
2. Earned restraint: use color only for state, risk, sentiment, selection, and primary action.
3. Trust through clarity: labels, metrics, charts, and report sections must be immediately legible.
4. Density with hierarchy: show enough operational information without making every element compete.
5. One system vocabulary: navigation, buttons, cards, forms, chips, and chart panels should feel from the same product.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Do not rely on color alone for risk or sentiment. Preserve keyboard focus, use clear labels for forms and icon actions, keep touch targets at least 44px on mobile, and respect reduced motion for any transitions.
