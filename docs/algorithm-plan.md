# 网络舆情事件智能分析系统 —— 算法部分框架规划

> 负责范围：数据预处理、内容分析、热点事件发现、舆情事件聚合、情感倾向分析、
> 事件舆情预测，以及高级功能（虚假文本检测、事件溯源与传播路径）。
> 爬虫采集、前端界面、账号体系不在本文档范围内，仅在接口处约定输入输出。

> **接口现状**：本文档第 4 节的 `/analyze/*` 接口是早期规划草案。后端已在
> `api-design/`（`events.json`、`prediction.json` 等）给出具体的字段名和 JSON
> 形状，`algo/pipeline.py` 现在是照这份契约实现的（字段名如 `heat`/`keywords`/
> `platform_distribution`/`trend`/`stage` 等），以 `api-design/` 为准，本节仅作
> 历史参考。差异点见 `algo/README.md`"对齐后端 api-design 接口"一节。

## 1. 整体流水线

```
原始网页/帖子(爬虫产出)
        │
        ▼
① 数据预处理  (清洗/去重/正文抽取/格式标准化)
        │
        ▼
② 内容分析    (分词/关键词/NER/向量化表示)
        │
        ├────────────────────────────┐
        ▼                            ▼
③ 热点事件发现(话题分类+增量聚类)   ⑥ 情感倾向分析(正/负/中性)
        │                            │
        ▼                            │
④ 舆情事件聚合(同事件报道合并/相似检索)◄┘
        │
        ▼
⑤ 事件舆情预测(生命周期阶段+发展趋势)
        │
        ▼
⑦ 高级功能：虚假文本检测 / 传播路径溯源
        │
        ▼
   输出给后端存储 → 前端展示（看板、详情报告）
```

各阶段均设计为**独立可调用的模块**（函数或微服务接口），便于并行开发、单独测试和替换实现。

## 2. 模块详细设计

### ① 数据预处理
- **去重**：SimHash / MinHash + LSH，对标题+正文计算指纹，汉明距离阈值判重复采集。
- **去噪 & 正文抽取**：正文抽取算法（如 GeneralNewsExtractor / trafilatura 思路：基于文本密度/标签密度定位正文），去除导航栏、广告、评论区。
- **格式标准化**：统一字段 schema（`title, content, publish_time, source, platform, url, author`），时间统一为 ISO8601，编码统一 UTF-8。
- 输出：`Document` 标准结构，供后续模块消费。

### ② 内容分析
- 分词/词性标注：jieba（快速原型）或 HanLP/LTP（效果更好）。
- 关键词提取：TF-IDF 或 TextRank，用于"高频关键词"展示。
- 命名实体识别（NER）：抽取人物、地点、机构，用于事件摘要的"涉事人物/地点"。
- 向量化表示：
  - 轻量方案：TF-IDF 向量 / Word2Vec 均值向量。
  - 效果更好：中文预训练模型句向量（如 Sentence-BERT / chinese-roberta-wwm-ext 的 [CLS] 向量），用于相似度计算和聚类。

### ③ 热点事件发现
- **话题分类**（可选，用于领域打标签）：有监督分类，TextCNN 或 BERT 微调。
- **事件聚类（核心）**：流式数据不适合一次性聚类，建议用 **Single-Pass 增量聚类**：
  - 新文档到来时，与已有事件簇的中心向量计算余弦相似度；
  - 若最大相似度超过阈值，归入该簇并更新簇中心；否则新建事件簇。
- **热度指数计算**：综合报道数量、时间衰减（牛顿冷却/指数衰减）、互动量（转/评/赞，如可采集），归一化为热度分值，用于看板排序。

### ④ 舆情事件聚合
- 同一事件的多条报道即为③中同一簇内的文档，做去重合并、来源列表汇总。
- 历史事件检索：对事件簇中心向量建索引（FAISS），支持"相似历史事件"检索。
- 事件摘要：抽取式摘要（TextRank 选取代表句）生成"事件概述"，或调用大模型做摘要/问答（对应前端"智能问答"功能）。

### ⑤ 情感倾向分析
- 主方案：中文预训练模型微调三分类（正面/负面/中性），如 bert-base-chinese + 分类头，可用公开中文情感数据集（如微博情感数据集）预训练/微调。
- 兜底方案：情感词典法（如 SnowNLP / 大连理工情感词典）做无标注场景的快速上线和结果校验。
- 输出：单文档情感标签+置信度 → 聚合为事件级情感分布（对应前端饼图/词云）。

### ⑥ 事件舆情预测（生命周期）
- 用事件簇内报道量随时间的序列作为观测值。
- **阶段判断**：拟合传播曲线（Logistic 增长模型或 SIR 类模型），依据曲线斜率/二阶导数变化，划分：
  - 潜伏期：报道量低且平稳
  - 成长期：报道量加速上升
  - 高潮期：报道量达到峰值附近，增速趋缓
  - 衰退期：报道量持续下降
- **关键时间节点标注**：变点检测算法（如 PELT，`ruptures` 库）定位曲线突变点，标注在发展趋势图上。
- 进阶：ARIMA/Prophet 或简单 LSTM 做短期报道量预测（选做）。

### ⑦ 高级功能
- **虚假文本检测**：
  - 特征：文本情感强度、来源可信度（来源历史准确率）、是否有权威信息源交叉报道、发布账号历史行为。
  - 模型：BERT 文本分类微调（真/谣言二分类），可用公开谣言检测数据集训练迁移；输出判断+置信度。
- **事件溯源与传播路径**：
  - 构建传播图：节点=账号/媒体，边=转发/引用/报道关系，按时间排序。
  - 关键节点识别：图算法（PageRank / 介数中心性，`networkx` 实现）定位首发账号、首次大V转发、首次官方媒体介入。
  - 输出节点+边的图数据结构，交由前端渲染传播路径图。

## 3. 技术栈建议

| 层面 | 选型 |
|---|---|
| 语言 | Python |
| 分词/NLP基础 | jieba / HanLP |
| 深度学习 | PyTorch + HuggingFace Transformers（中文BERT/RoBERTa） |
| 向量检索 | FAISS |
| 时间序列/变点检测 | statsmodels / ruptures / (可选)Prophet |
| 图算法 | networkx |
| 服务化 | FastAPI 封装为算法微服务，REST 接口供后端调用 |

## 4. 对外接口约定（与后端联调用）

- `POST /analyze/preprocess` 原始文档 → 标准化文档
- `POST /analyze/nlp` 文档 → 分词/关键词/向量
- `POST /analyze/cluster` 文档流 → 事件簇ID
- `POST /analyze/sentiment` 文档 → 情感标签+置信度
- `GET  /analyze/hotness` 事件簇 → 热度分值
- `GET  /analyze/lifecycle` 事件簇 → 生命周期阶段 + 趋势数据点
- `POST /analyze/factcheck` 文档 → 真实性判断+置信度（高级）
- `GET  /analyze/propagation` 事件簇 → 传播路径图数据（高级）

## 5. 开发里程碑建议

1. **M1 基础打通**：预处理 + 分词特征 + 报道量热度打分 + 词典法情感 → 让看板先跑起来。
2. **M2 核心算法**：Single-Pass 事件聚类聚合 + BERT情感分类 + 关键词/摘要。
3. **M3 趋势预测**：生命周期阶段判断 + 变点检测 + 发展趋势图数据。
4. **M4 高级功能**：虚假文本检测 + 传播路径溯源与可视化数据。
5. **M5 联调与评估**：与前后端联调；用 Accuracy/F1（分类）、NMI/ARI 或人工抽样（聚类）等指标评估算法效果，整理成报告素材。

## 6. 评估指标参考

- 情感分类 / 话题分类 / 谣言检测：Accuracy、Precision/Recall、F1
- 事件聚类：NMI、ARI（若有标注）或人工抽样评估聚类纯度
- 生命周期/趋势预测：与真实报道量曲线的拟合误差（RMSE），关键节点标注的人工合理性检查
