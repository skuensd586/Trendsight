# 前端功能与后端接口补齐说明

## 1. 事件看板列表

前端缺少搜索功能和排序功能实现
必须补全的基础功能：时间热度排序
可以补全功能：搜索功能

| 前端参数 | 用途 | 后端当前状态 |
|---|---|---|
| `q` | 搜索事件标题、关键词、分类、地点 | 未实现 |
| `risk_level` | 风险等级筛选 | 未实现 |
| `time_range` | 时间范围筛选 | 未实现 |
| `sort` | 排序：热度、时间、负面情绪 | 未实现 |

## 2.事件详情页
必须补全的基本功能：缺少事件概述，直接起因，地点，涉事主体



前端详情页还需要但后端缺少：

| 缺失字段 | 前端用途 |
|---|---|
| `summary` | 事件概述 （必做）|
| `category` | 分类展示（可不做） |
| `location` | 地点（必做） |
| `cause` | 起因 （必做）|
| `people` | 涉事主体（必做） |
| `advice` | 处置建议 （可不做）|
| `duplicate_rate` | 重复传播率（转发搬运的比例） |
| `similar_events` | 历史相似事件（可不做） |
| `traceability.nodes` | 溯源节点（可不做） |
| `traceability.links` | 溯源关系 （可不做）|
| `geo_discussion` | 地域热力图（可不做） |
| `authenticity.false_confidence` | 真实性置信度 （可不做的高级功能）|
| `authenticity.checks` | 真实性检测说明（可不做的高级功能） |



## 3. 智能问答

后端当前状态：

| 功能 | 状态 |
|---|---|
| 事件问答 | 已实现 |
| 对话历史 | 已实现 |
| 多轮 `conversation_id` | 已实现 |
| 基于事件详情增强回答 | 需要确认后端是否读取事件数据 |
| LLM Key 配置 | 依赖 `.env` |

请求字段：

| 字段 | 用途 |
|---|---|
| `event_id` | 当前事件 ID |
| `conversation_id` | 多轮对话 ID，可为空 |
| `question` | 用户问题 |

返回字段：

| 字段 | 用途 |
|---|---|
| `conversation_id` | 保存多轮对话 |
| `answer` | 大模型回答 |
| `created_time` | 回答时间 |

## 7. 个人中心配置
后端无法支持采集频率和采集源状态相关的字段（但可以不要这个功能）


后端当前已支持：

| 字段 | 用途 |
|---|---|
| `fields` | 关注领域 |
| `keywords` | 关注关键词 |
| `platform_urls.platform_name` | 平台名称 |
| `platform_urls.url` | 平台地址 |

前端传了但后端未持久化：

| 前端字段 | 用途 |
|---|---|
| `frequency_minutes` | 采集频率 |
| `status` | 采集源状态：正常 / 限流 |


## 8. 爬虫与分析触发

当前前端没有直接触发爬虫的功能。

当前后端也没有对前端暴露：

（这个功能也不是必须的）

建议后端补充：

| 接口 | 用途 |
|---|---|
| `POST /api/crawler/run` | 手动触发爬虫 |
| `POST /api/analysis/run` | 手动触发 raw 数据分析 |
| `GET /api/analysis/status` | 查看分析任务状态 |
| `GET /api/crawler/status` | 查看爬虫运行状态 |
